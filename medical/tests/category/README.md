# Тесты Рентгенолаборанта - Руководство

## 📋 Что это?

Веб-приложение для бесконечной тренировки знаний по медицинским тестам. Работает полностью на клиентской стороне, не требует сервера или базы данных.

## 🔄 Как добавить новые вопросы

### Способ 1: Прямое редактирование через GitHub (Рекомендуется)

1. **Откройте файл questions.txt** в вашем репозитории на GitHub
2. **Нажмите кнопку редактирования** (иконка карандаша)
3. **Прокрутите в конец файла**
4. **Добавьте новые вопросы** в следующем формате:

```
Q: Ваш новый вопрос?
A: Первый вариант ответа
B: Второй вариант ответа
C: Третий вариант ответа
D: Четвертый вариант ответа
CORRECT: A

[обязательно пустая строка между вопросами]
```

5. **Сохраните изменения** (Commit changes)
6. **Готово!** Обновите страницу в браузере (Ctrl+F5 или Cmd+Shift+R), и новые вопросы появятся

### Способ 2: Редактирование на компьютере

1. **Скачайте questions.txt** из репозитория
2. **Откройте в любом текстовом редакторе** (Notepad++, VS Code, Sublime Text)
   - ⚠️ Важно: НЕ используйте Microsoft Word!
   - Используйте именно текстовый редактор
3. **Добавьте новые вопросы в конец файла**
4. **Сохраните файл в кодировке UTF-8**
5. **Загрузите обновленный файл** обратно на GitHub
6. **Обновите страницу** в браузере

### Способ 3: Массовое добавление из Excel/CSV

Если у вас много новых вопросов в таблице:

1. **Подготовьте CSV файл** с колонками:
   - Вопрос | Вариант_A | Вариант_B | Вариант_C | Вариант_D | Правильный

2. **Используйте скрипт конвертации:**
   ```bash
   python convert_questions.py новые_вопросы.csv новые.txt
   ```

3. **Откройте оба файла** (старый questions.txt и новые.txt)

4. **Скопируйте содержимое новые.txt** и **вставьте в конец questions.txt**

5. **Загрузите обновленный questions.txt** на GitHub

### ⚠️ Важные правила при добавлении вопросов:

✅ **ОБЯЗАТЕЛЬНО** оставляйте пустую строку между вопросами  
✅ **ОБЯЗАТЕЛЬНО** сохраняйте в кодировке UTF-8  
✅ Не используйте лишние пробелы в конце строк  
✅ Буквы вариантов: A, B, C, D, E и т.д. (заглавные латинские)  
✅ Для нескольких правильных ответов: `CORRECT: A,B,C` (без пробелов!)

### Пример добавления нового вопроса:

**Было (конец файла):**
```
Q: Предыдущий вопрос?
A: Ответ 1
B: Ответ 2
CORRECT: A
```

**Стало:**
```
Q: Предыдущий вопрос?
A: Ответ 1
B: Ответ 2
CORRECT: A

Q: Новый вопрос о рентгене?
A: Первый вариант
B: Второй вариант
C: Третий вариант
CORRECT: B
```

## 📝 Формат файла questions.txt

### Базовая структура:

```
Q: Текст вопроса?
A: Первый вариант ответа
B: Второй вариант ответа
C: Третий вариант ответа
D: Четвертый вариант ответа
CORRECT: A

[пустая строка между вопросами]

Q: Следующий вопрос?
...
```

### Правила:

1. **Вопрос** начинается с `Q:`
2. **Варианты ответов** с `A:`, `B:`, `C:`, `D:`, `E:` и т.д.
3. **Правильные ответы** с `CORRECT:`
   - Один правильный: `CORRECT: A`
   - Несколько правильных: `CORRECT: A,C,D`
4. **Между вопросами** обязательно пустая строка

### Примеры:

**Один правильный ответ:**
```
Q: Какой орган вырабатывает инсулин?
A: Печень
B: Поджелудочная железа
C: Щитовидная железа
CORRECT: B
```

**Несколько правильных ответов:**
```
Q: Признаки инфаркта (может быть несколько)?
A: Боль в груди
B: Высокая температура
C: Одышка
D: Холодный пот
CORRECT: A,C,D
```

## 🔄 Как добавить свои вопросы

### ⚡ ВАЖНО: Два способа работы с вопросами

Из-за ограничений CORS на GitHub Pages, есть два подхода:

#### **Вариант 1: Встроенные вопросы (РЕКОМЕНДУЕТСЯ)**

Вопросы встраиваются прямо в JavaScript файл. Это самый надежный способ.

**Шаги:**

1. **Отредактируйте файл `questions.txt`** в нужном формате (см. ниже)

2. **Запустите скрипт конвертации:**
   ```bash
   python txt_to_js.py
   ```
   Это создаст файл `questionsData.js`

3. **Добавьте в `index.html` перед строкой `<script src="app.js"></script>`:**
   ```html
   <script src="questionsData.js"></script>
   <script src="app.js"></script>
   ```

4. **Загрузите файлы на GitHub:**
   - index.html (обновленный)
   - app.js
   - style.css
   - questionsData.js (новый файл)

**Преимущества:**
- ✅ Работает гарантированно
- ✅ Быстрая загрузка
- ✅ Нет проблем с CORS

#### **Вариант 2: Внешний файл questions.txt**

Если вариант 1 не подходит, можно использовать внешний файл.

**Требования:**
- Файл `questions.txt` должен быть в той же папке
- Может не работать при локальном просмотре (file://)
- Может потребоваться настройка CORS на сервере

Приложение автоматически попробует загрузить `questions.txt`, если он есть. Если загрузка не удастся - будут использованы встроенные вопросы.

### 📝 Формат файла questions.txt

1. **Структурируйте их в следующем виде** (например, в Excel):
   - Колонка A: Вопрос
   - Колонка B: Вариант A
   - Колонка C: Вариант B
   - Колонка D: Вариант C
   - Колонка E: Вариант D
   - Колонка F: Правильные ответы (A или A,C,D)

2. **Используйте формулу для конвертации** (в Excel):
   ```
   ="Q: "&A2&CHAR(10)&"A: "&B2&CHAR(10)&"B: "&C2&CHAR(10)&"C: "&D2&CHAR(10)&"D: "&E2&CHAR(10)&"CORRECT: "&F2&CHAR(10)
   ```

3. **Или используйте Python скрипт** (сохраните как `convert.py`):
   ```python
   # Пример: если ваши вопросы в CSV
   import csv
   
   with open('questions.csv', 'r', encoding='utf-8') as csvfile:
       reader = csv.DictReader(csvfile)
       with open('questions.txt', 'w', encoding='utf-8') as output:
           for row in reader:
               output.write(f"Q: {row['Вопрос']}\n")
               output.write(f"A: {row['Вариант_A']}\n")
               output.write(f"B: {row['Вариант_B']}\n")
               output.write(f"C: {row['Вариант_C']}\n")
               if row.get('Вариант_D'):
                   output.write(f"D: {row['Вариант_D']}\n")
               output.write(f"CORRECT: {row['Правильный']}\n\n")
   ```

4. **Запустите:** `python convert.py`

## ✨ Функции приложения

### Основные возможности:

✅ **Бесконечный режим тестирования** - тренируйтесь столько, сколько хотите

✅ **Умная система повторений** - вопросы, на которые вы ответили неправильно, будут появляться чаще

✅ **Частично правильные ответы** - если вы выбрали не все правильные варианты или добавили неправильные, система покажет это оранжевым цветом

✅ **Статистика:**
- Общая статистика по всем вопросам
- Статистика текущей сессии
- Детальная статистика по каждому вопросу
- Список самых сложных вопросов

✅ **Адаптивный дизайн** - работает на iPhone, Android планшете, ПК

✅ **Светлая/темная тема** - переключается кнопкой в правом нижнем углу

✅ **Сохранение прогресса** - все данные хранятся локально в браузере

✅ **Клавиатурные shortcuts:**
- Enter - проверить/следующий вопрос
- 1-9 - выбрать вариант ответа

### Цветовая индикация:

- 🟢 **Зеленый** - полностью правильный ответ
- 🟠 **Оранжевый** - частично правильный ответ (есть правильные варианты, но не все или есть лишние)
- 🔴 **Красный** - неправильный ответ

### Как это работает:

1. **Нажмите "Начать тестирование"** на стартовом экране
2. **Отвечайте на вопросы** - выбирайте один или несколько вариантов
3. **Проверьте ответ** - кнопка "Проверить"
4. **Смотрите результат:**
   - ✅ Зеленый = полностью правильно
   - ⚠️ Оранжевый = частично правильно (покажет полный правильный ответ)
   - ❌ Красный = неправильно (покажет правильный ответ)
5. **Следующий вопрос** - и так до бесконечности
6. **Завершите тест** нажав крестик (✕) в правом верхнем углу
   - Подтвердите завершение
   - Весь прогресс сохранится автоматически
7. **Анализируйте статистику** - найдите слабые места

## 🎯 Алгоритм умного повтора

Каждому вопросу присваивается "вес" в зависимости от вашей успеваемости:

- **Новый вопрос** = вес 1 (базовый)
- **Правильный ответ** = вес уменьшается
- **Неправильный ответ** = вес увеличивается

Вопросы с большим весом выпадают чаще. Это помогает сфокусироваться на сложных темах.

**Важно:** Данные привязаны к браузеру. Если вы используете разные устройства или браузеры, прогресс будет отдельный.

## 🐛 Устранение неполадок

### Вопросы не загружаются:

**Если видите "Ошибка загрузки вопросов":**

1. **Используйте встроенные вопросы (Вариант 1):**
   - Отредактируйте `questions.txt`
   - Запустите `python txt_to_js.py`
   - Добавьте `<script src="questionsData.js"></script>` в HTML
   - Загрузите `questionsData.js` на GitHub

2. **Проверьте формат `questions.txt`:**
   - Кодировка UTF-8
   - Правильный формат (Q:, A:, B:, CORRECT:)
   - Пустые строки между вопросами

3. **Откройте консоль браузера (F12 → Console):**
   - Посмотрите на сообщения об ошибках
   - Должно быть: "Загружено вопросов: N"

4. **Проверьте структуру папок:**
   ```
   medical/tests/category/
   ├── index.html
   ├── style.css
   ├── app.js
   └── questionsData.js  (или questions.txt)
   ```

### Прогресс не сохраняется:

1. Проверьте, что браузер позволяет использовать localStorage
2. Не используете ли вы режим инкогнито?
3. Проверьте настройки конфиденциальности браузера

### Не работает на мобильном:

1. Очистите кэш браузера
2. Обновите страницу
3. Проверьте подключение к интернету (для первой загрузки)
